# SFTP 文件传输详细指南

## 远程路径规则

### 必须使用绝对路径

所有 `file_transfer.py` 操作的远程路径必须使用绝对路径。

- 正确：`/home/user/file.deb`
- 错误：`~/file.deb`、`$HOME/file.deb`、`./file.deb`

> 注：`v2` 版本已新增自动路径解析，`~` 开头的路径会被自动展开为绝对路径，但推荐始终使用绝对路径以避免歧义。

### 路径解析机制

`file_transfer.py` 混用 SSH shell 和 SFTP 两种机制：
- `upload` 通过 shell 管道写入（shell 会展开 `~`）
- `download`/`info`/`ls` 通过 SFTP 协议（SFTP 不展开 `~`，当作字面路径）

为保证所有操作行为一致，代码已内置 `resolve_remote_path()` 函数，自动将 `~` 和相对路径解析为绝对路径。

### 获取远程 home 目录

如需确认远程 home 目录的绝对路径：

```bash
python scripts/ssh_execute.py execute -H <host> -u <user> -c "echo $HOME"
```

## 完整参数说明

### 通用参数

| 参数 | 短选项 | 说明 | 默认值 |
|------|--------|------|--------|
| `--host` | `-H` | 目标主机 | （必填）|
| `--port` | `-P` | SFTP/SSH 端口 | 22 |
| `--user` | `-u` | 用户名 | （必填）|
| `--password` | `-p` | 密码 | 交互输入 |
| `--key` | `-k` | 私钥路径 | 无 |
| `--retry` | | 重试次数 | 3 |
| `--audit-log` | | 审计日志路径 | audit_remote.log |

### upload 专有参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--local` / `-l` | 本地文件路径（必填）| |
| `--remote` / `-r` | 远程路径，推荐绝对路径（必填）| |
| `--no-md5` | 禁用 MD5 校验 | 关闭 |

### download 专有参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--remote` / `-r` | 远程文件路径，推荐绝对路径（必填）| |
| `--local` / `-l` | 本地保存路径（必填）| |
| `--no-md5` | 禁用 MD5 校验 | 关闭 |

### info 专有参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--path` | 远程路径，推荐绝对路径（必填）| |

### ls 专有参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--path` | 远程目录，推荐绝对路径（必填）| |
| `--page` | 页码 | 1 |
| `--page-size` | 每页条数 | 100 |

## 认证方式

### 密钥认证

```bash
python scripts/file_transfer.py upload -H <host> -u <user> -k ~/.ssh/id_rsa -l <local> -r /home/user/<remote>
```

### 交互式密码

不提供 `--password` 和 `--key` 时自动提示输入密码。

## 安全说明

- 文件上传/下载操作会记录审计日志（`UPLOAD`/`DOWNLOAD` 动作）
- 文件传输不触发命令级安全拦截（因为文件传输不等同 shell 命令执行）
- 审计日志默认写入 `audit_remote.log`，可通过 `--audit-log` 自定义路径

## 常见错误与排查

| 错误现象 | 原因 | 解决方法 |
|----------|------|----------|
| "No such file" 但文件确实存在 | 路径使用了 `~` 或相对路径 | 改用绝对路径 |
| 连接被拒绝 | 端口或防火墙 | 检查 `--port`，确认 SSH 服务运行 |
| MD5 不匹配 | 传输中断 | 重试传输 |
| 权限不足 | 远程目录写权限 | 检查目标目录权限 |
| 认证失败 | 密钥/密码错误 | 确认 `--key` 路径或密码 |
| 路径含空格失败 | shell 解析问题 | 使用绝对路径，代码已自动转义 |
